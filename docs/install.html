Install
& connect

Get the minipainter CLI, then hand your bench to any MCP-capable agent. Every path below, start to finish. Node 18 or newer, that is the only prerequisite.

1 · Get the CLI

Run it straight from npm with npx, no clone, no setup. The 1,607 paint catalog is bundled, so search and matching work on the first run.

# run straight from npm
$ npx minipainter paint search bone
$ npx minipainter match color "#d2c29b"
$ npx minipainter tui

Or install it globally to get the short mpaint command everywhere:

# install once, call from anywhere
$ npm install -g minipainter
$ mpaint paint search bone

Prefer to work from source? Clone and run against the working tree:

# hack on it locally
$ git clone https://github.com/arturskowronski/warpaint-cli
$ cd warpaint-cli && npm install
$ node src/cli.mjs paint search bone

Your inventory lives at ~/.minipainting/inventory.json, created automatically the first time you mark a paint owned. A legacy ~/.warpaint/ is migrated on first run.

2 · Connect an agent

minipainter speaks the Model Context Protocol, so it plugs into any agent that does. Pick the one you use.

Claude Desktop · local

Local MCP server

Add minipainter to claude_desktop_config.json, then restart Claude Desktop. It reads and updates your local inventory directly.

// claude_desktop_config.json
{
  "mcpServers": {
    "minipainter": {
      "command": "npx",
      "args": ["minipainter-mcp"]
    }
  }
}

From a clone, use "command": "node" with the absolute path to src/mcp-server.mjs.

Claude web · ChatGPT · Gemini

Hosted remote MCP

No install. Add the hosted endpoint as a custom connector in the client's settings and browse the catalog in chat. Claude web and Gemini use /mcp; ChatGPT uses the search / fetch route at /mcp/v3.

https://warpaint-mcp.fly.dev/mcp
https://warpaint-mcp.fly.dev/mcp/v3

The hosted inventory is shared; self-host below to keep your own.

Claude Code · skill

Agent Skill

The repo ships a standards-compliant Agent Skill. Drop it into your project or your home config and Claude Code drives the CLI with the right guardrails.

# make it available in a project
$ cp -r warpaint-cli/.claude/skills/minipainter \
     your-project/.claude/skills/

Or copy it to ~/.claude/skills/ to use it everywhere.

3 · Self-host the hosted endpoint

Want your own remote MCP with a private inventory rather than the shared one? Bring up the server and Postgres with Docker, or deploy to Fly.

# from the repo, server + Postgres
$ docker compose up -d
$ curl localhost:8787/mcp

Set DATABASE_URL to move inventory into Postgres; without it, inventory stays in a JSON file. Fly steps live in docs/deploy-fly.md in the repo.
